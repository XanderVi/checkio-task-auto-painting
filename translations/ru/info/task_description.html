Никола собрал авто-покрасочную систему, но она может красить только одну сторону деталей за раз. После этого оператор должен перезапустить машину и покрасить другую сторону (система выбирает неокрашенную сторону автоматически). Один цикл покраски всегда происходит за одно и тоже время. Камера вмещает K деталей за раз. Никола хочет, чтобы Стефан работал с этим аппаратом и ему нужен алгоритм, который позволит покрасить N деталей за наименьшее число циклов. Главное не покрасьте деталь лишний раз - два раза и все.

Детали пронумерованы от 0 до 9. Дана вместимость покрасочной камеры (K) и количество деталей (N). Необходимо найти последовательность действий, которую Стефан будет использовать, как строку. Каждое действие это номера деталей для покраски в данном цикле. Действия разделяются запятой (",").

Входные данные: Вместимость покрасочной камеры и число деталей, как целые числа (int).

Выходные данные: Последовательность действий, как строка (str).

Примеры: 

checkio(2, 3)  # "01,12,02"
checkio(6, 3)  # "012,012"
checkio(3, 6)  # "012,012,345,345"
checkio(1, 4)  # "0,0,1,1,2,2,3,3"
checkio(2, 5)  # "01,01,23,42,34"

Зачем это нужно: По сути - это классическая задача о том как поджарить три котлеты на сковороде для двух котлет. Здесь мы расширили эту задачу и переложили на технический лад. Таким образом можно моделировать и оптимизировать реальные процессы на производстве.

Предусловия:
0 < capacity ≤ 10
0 < number ≤ 10
