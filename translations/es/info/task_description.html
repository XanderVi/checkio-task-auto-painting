Nicola ha construido un sistema de pintura semi-automática. Sin embargo, este sistema solo puede pintar un lado de un elemento, por lo que cada vez, es necesario que un operador recargue la máquina y pinte el otro lado (el sistema detecta los lados pintados de forma automática). El proceso de pintura siempre toma la misma cantidad de tiempo. La cámara puede pintar K superficies a la vez. Nicola quiere que Stephan opere la máquina de pintar y necesita desarrollar un algoritmo para Stephan que le permita pintar N (0

Los elementos se numeran de 0 a 9. Se te informará la capacidad de almacenamiento de pintura de la máquina (K) y la cantidad de artículos (N) a pintar. Deberás encontrar la secuencia que Stephen debe pintar como una cadena (str), donde cada acción es el número de elementos pintados. Las acciones están separadas por comas (",").

Datos de Entrada: Capacidad del sistema de pintura y cantidad de artículos a pintar, como enteros (int).

Salida: La secuencia de acciones, como una cadena (str).

¿Cómo se usa?: Esto es similar a cocinar tres bistecs en una sartén con capacidad para dos. Todos los bistecs tienen dos lados y se tarda un minuto para cocinar un lado de los bistecs, ¿cómo cocinar todos los filetes en tres minutos? Esta tarea utiliza este concepto y lo modela de una manera técnica. Las ideas detrás de esta misión también pueden modelar un proceso tecnológico real en fábricas.
